# Hebrew Microcopy — DocuMed

## Tone guidelines

- **חם, לא קליני** — "שמרנו בהצלחה" לא "הפעולה הושלמה"
- **ברור, לא מדעי** — בגובה העיניים, לא שפה רפואית
- **מרגיע, לא מפחיד** — שגיאות לא מטרידות, הצלחות מעודדות
- **קצר** — כפתורים עד 3 מילים, הודעות עד 2 משפטים
- **פעיל** — "הוסף תרופה" לא "הוספת תרופה"
- **גוף שני מכבד** — "תוכל" / "נסה" / "בחר"

## Step 1 — Understand the feature

Ask the user (if not provided):
1. **פיצ'ר** — על מה הקופי? (לדוגמה: "מחיקת תרופה", "שגיאת שמירה", "מסך ריק בלי מחלות")
2. **מצבים נדרשים** — אילו מצבים צריך קופי? (ראה רשימה למטה)
3. **הקשר רפואי** — האם יש מונח רפואי ספציפי שצריך להופיע?

## Step 2 — Write copy for each state

### מצבים סטנדרטיים לכל פיצ'ר

| מצב | מה לכתוב |
|---|---|
| **כותרת** | שם הפיצ'ר — ברור, קצר, שם עצם |
| **כפתור ראשי** | פועל + מושא — "הוסף תרופה", "שמור שינויים" |
| **כפתור ביטול** | "ביטול" / "חזרה" — לא "לא" |
| **מסך ריק** | משפט חם + הנעה לפעולה |
| **טוען** | "רגע..." / "טוען את הנתונים..." |
| **הצלחה** | קצר, חם — "נשמר!" / "נמחק בהצלחה" |
| **שגיאה** | מה קרה + מה לעשות — ללא מילה "שגיאה" |
| **אישור מחיקה** | "בטוח?" + תיאור מה יימחק + "כן, מחק" / "ביטול" |
| **Placeholder** | דוגמה ריאלית — "לדוגמה: מטפורמין 500 מ״ג" |
| **Tooltip** | משפט אחד — מה הפעולה עושה |

### דוגמאות לעיצוב קופי

**מסך ריק — רשימת תרופות:**
> "עדיין לא נוספו תרופות.
> לחץ על ״הוסף תרופה״ כדי להתחיל."

**הצלחה — שמירת תרופה:**
> "התרופה נשמרה בהצלחה."

**שגיאה — לא ניתן לשמור:**
> "לא הצלחנו לשמור את הנתונים. נסה שוב, ואם הבעיה חוזרת — רענן את הדף."

**אישור מחיקה:**
> "למחוק את אמוקסיצילין 500 מ״ג?
> לא ניתן לשחזר אחרי המחיקה."
> [כן, מחק] [ביטול]
